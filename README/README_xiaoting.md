# 这是肖婷的规划代码思路

本路径规划节点整体逻辑分为锥桶融合预处理 → 锥桶筛选分类 → 全局最优中心线生成 → 轨迹平滑优化 → 弯道延伸补全 → 轨迹输出六步，全程针对直角弯高曲率、锥桶不对称、视野截断问题专项优化。

## 一、关键算法
### 1：全局代价锥桶配对算法
**传统方案缺陷**：常规规划采用左右锥桶下标一一配对，直角弯内外侧锥桶分布不均、前后错位，极易出现跨区域错配，导致赛道中心线严重偏移，车辆过弯推头、跑偏。
**此代码核心逻辑**：放弃机械下标配对，采用**全局遍历\+代价权重筛选\+一对一最优匹配**机制，完全适配直角弯非对称锥桶分布：

1\. 合法性过滤
通过双层遍历生成所有锥桶配对候选，剔除错位、超宽异常配对，适配直角弯狭窄、高曲率特性：
- `if x_gap > 2.5: continue` 过滤前后纵向错位过大的锥桶，禁止弯道跨段错配
- `if track_width < 1.0 or track_width > 6.0: continue` 过滤异常赛道宽度，规避弯道离群锥桶干扰

2\. 专属弯道代价函数
`score = 2.0 * x_gap + 0.3 * abs(track_width - default_width)`
源码**大幅提升纵向错位权重（2\.0）**，远大于赛道宽度误差权重，优先保证同一横截面的左右锥桶配对，从算法根源解决直角弯错配问题。

3\. 全局唯一匹配机制
源码通过 `used_left、used_right` 集合记录已使用锥桶，排序后贪心匹配，保证每个锥桶仅参与一次配对，无重复、无漏配。


### 2：分段差异化高斯平滑

**传统方案缺陷**：原始锥桶生成的轨迹为离散折线，直角弯拐点尖锐、曲率不连续，导致转向角度剧烈跳变，车身抖动、过弯稳定性极差。一刀切平滑方案会损耗直道精度、弯道平滑效果不足。

**此代码核心逻辑**：实现**直道保精度、弯道强平滑**的差异化自适应处理，专属优化直角弯轨迹质感：

1\. 弧长均匀密化
源码通过弧长插值算法，将稀疏锥桶点列密化为 `step=0.12m` 均匀稠密轨迹，为弯道平滑提供充足数据基础。

2\. 曲率场景判别
设置阈值 `curve_threshold = 0.03`，自动区分路况：曲率低于阈值判定为直道，保留原始轨迹保证行驶精度；曲率高于阈值（直角弯）启动专属平滑逻辑。

3\. 弯道高斯加权平滑
对直角弯高曲率区域，采用 `window_size=5` 窗口高斯加权移动平均，通过正态分布权重磨平尖锐拐点，替代生硬折线轨迹。


### 3：弯道自适应圆弧延伸

**传统方案缺陷**：直角弯远端锥桶易超出感知视野，规划轨迹提前截断，常规直线延伸逻辑会强行拉直弯道轨迹，引导车辆直线冲出赛道，是直角弯失控的核心原因。
**此代码核心逻辑**：首创**弯道圆弧延伸、直道线性延伸**的自适应分支逻辑，针对性解决盲弯轨迹缺失问题：

1\. 延伸触发机制
`if total_len > 8.0: return points` 轨迹长度不足8m（盲弯、短弯道）自动触发延伸，默认延伸长度4m、步长0\.15m。
2\. 直角弯专属圆弧延伸（核心亮点）
源码通过末端5个轨迹点拟合局部曲率，计算弯道半径R、圆心坐标，根据叉积判断弯道偏转方向，**沿圆弧切向增量生成延伸点**，而非简单直线补点，完美贴合直角弯走向。
3\. 通用兼容逻辑
低曲率直道自动降级为线性延伸，兼顾全场景行驶需求。


## 二、辅助机制

### 1\. 多帧锥桶时序融合滤波

源码设置`cone_max_age = 8.0s` 过期清理机制，剔除历史无效锥桶；对0\.5m内同位置同颜色锥桶，采用 `0.8*历史坐标+0.2*新坐标` 加权融合，抑制感知单帧抖动，弥补直角弯单侧锥桶漏检问题，保证赛道边界稳定。

### 2\. 全流程ODOM全局坐标统一

源码通过TF变换，将车身base\_link局部坐标系锥桶，统一转换至odom全局坐标系，全程规划基于固定全局坐标系计算，杜绝车辆转弯时坐标旋转漂移，保证直角弯轨迹全程稳定不偏移。

## 三、所有函数的概述
1. 锥桶时序融合与坐标统一（merge_known_cones）
节点订阅感知锥桶数据，通过TF将车身局部坐标统一转换为odom全局坐标；维护全局锥桶缓存与时间戳，剔除8秒过期无效锥桶，对近距离同色锥桶进行加权均值滤波融合，抑制感知抖动，弥补直角弯单侧锥桶漏检问题，保证赛道边界稳定。
2. 锥桶筛选与有序分类（plan_path_from_cones）
读取车辆全局位姿，将所有锥桶转换至车体局部坐标系，过滤车后无效锥桶；按颜色区分左右赛道边界，并根据车辆前向坐标排序，保证锥桶沿行驶方向有序排列，为弯道精准配对提供基础。
3. 全局代价最优锥桶配对，生成赛道中心线（pair_cones_to_centerline）
放弃传统下标配对方式，遍历生成所有合法左右锥桶配对，通过纵向错位、赛道宽度阈值过滤异常组合；设计侧重纵向对齐的代价函数，优先匹配同一横截面锥桶，再通过全局一对一贪心匹配，解决直角弯锥桶数量不对称导致的错配、中心线偏移问题，生成精准原始赛道中线。
4. 分段自适应轨迹平滑（smooth_path）
对原始中心线进行弧长均匀密化，提升轨迹稠密性；逐点计算轨迹曲率，区分直道与弯道，直道保留原始精度，直角弯等高曲率区域启用高斯加权滑动平均平滑，磨平折线拐点，实现轨迹曲率连续，消除过弯转向抖动。
5. 自适应圆弧路径延伸（extend_path）
针对直角弯远端锥桶出视野、轨迹提前截断的问题，短轨迹自动触发延伸；通过末端轨迹点拟合局部曲率，识别弯道姿态，高曲率直角弯沿圆弧切线方向延伸轨迹，低曲率直道采用线性延伸，补全盲区弯道轨迹，避免车辆直冲冲出赛道。
6. 轨迹封装与稳定发布（timer_callback）
计算平滑轨迹逐点航向角与曲率，封装为标准odom坐标系的Path消息，定时循环发布轨迹，供给MPC控制器跟踪，同时发布全局路径供SLAM模块使用，实现直角弯全程稳定、平顺、无冲出的自主轨迹规划。
